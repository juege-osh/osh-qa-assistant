# AI Assistant 课程作业与练习题清单

## 1. 文档定位

这份文档给你两种用途：

- 录课时布置课后作业
- 做训练营或陪跑时安排练习

目标不是出很学术的题，而是让学员真的把项目跑明白、改明白、讲明白。

## 2. 入门作业

### 2.1 作业一：本地完整跑通项目

要求：

- 安装 JDK 17
- 安装 Node 18
- 准备 MySQL、Redis、Qdrant
- 启动本地项目
- 成功访问本地前后端地址

提交内容：

- 启动截图
- 后端 `UP` 截图

### 2.2 作业二：初始化管理员并登录

要求：

- 执行表结构脚本
- 执行初始化管理员脚本
- 用 `admin / 123456` 登录管理端

提交内容：

- 数据库截图
- 管理端登录成功截图

## 3. 业务作业

### 3.1 作业三：创建知识库并上传文件

要求：

- 新建 1 个知识库
- 上传至少 2 份文档
- 查询文件分页列表

提交内容：

- 知识库页面截图
- 文件列表截图

### 3.2 作业四：创建应用并绑定知识库

要求：

- 新建 1 个应用
- 绑定 1 个知识库
- 通过 `checkChatCondition` 验证聊天条件

提交内容：

- 应用页面截图
- 绑定成功截图

### 3.3 作业五：跑通一次完整对话

要求：

- 新建会话
- 发 1 个知识库内问题
- 发 1 个知识库外问题
- 对比回答差异

提交内容：

- 聊天页面截图
- 你的观察总结

## 4. 调试作业

### 4.1 作业六：打断点看登录流程

要求：

- 在 `getCode`
- `login`
- `AuthorizationInterceptor.preHandle`

这 3 个位置打断点，讲清楚验证码、token 和鉴权流程。

提交内容：

- 断点截图
- 300 字解释

### 4.2 作业七：打断点看 RAG 主链路

要求：

- 在 `AiChatServiceImpl` 打断点
- 观察问题召回结果
- 观察增强后的 prompt

提交内容：

- 调试变量截图
- 你对 RAG 流程的文字说明

## 5. 接口作业

### 5.1 作业八：不用页面，手工调通接口

要求：

参考文档：

### 5.2 作业九：跑通开放 API

要求：

- 获取用户 `appKey`
- 调用 `/consumer/api/chat`
- 成功拿到 SSE 返回

提交内容：

## 7. 思考题

### 7.1 思考题一

为什么这个项目把 `manager` 和 `consumer` 分模块，但最终又统一到一个后端启动？

### 7.2 思考题二

为什么文件上传成功，不代表知识库就已经可以正确问答？

### 7.3 思考题三

为什么聊天接口要拆成“先连 SSE，再发聊天请求”两步？

### 7.4 思考题四

为什么开放 API 不直接复用用户登录密码，而是单独设计 `appKey`？

### 7.5 思考题五

为什么同样的代码，本地没问题，线上却可能出问题？

## 8. 进阶挑战题

### 8.1 挑战一：加一个模型切换配置

目标：

- 让系统支持不同 chat 模型切换

### 8.2 挑战二：加一个更细的权限校验

目标：

- 让部分资源访问边界更严格

### 8.3 挑战三：加一个更完整的调用审计页面

目标：

- 展示请求耗时、状态、失败原因、token 消耗等

## 9. 最适合你录课时布置的 3 个作业

如果你不想布置太多，最推荐的是：

1. 本地完整跑通项目
2. 不用页面手调一条登录和聊天接口
3. 在 `AiChatServiceImpl` 打断点讲清楚 RAG 链路

这 3 个作业最能区分“只是看懂页面”和“真正掌握项目”的差别。
