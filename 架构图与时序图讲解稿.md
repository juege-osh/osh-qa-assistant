# AI Assistant 架构图与时序图讲解稿

## 1. 文档定位

这份文档是给你做架构图、流程图、时序图时直接参考讲法的。

它不画图片，但会把图上应该有哪些元素、箭头怎么连、每一步怎么讲写清楚。

## 2. 项目总架构图该怎么画

建议画成 6 层：

1. 浏览器层
2. 前端应用层
3. 后端应用层
4. 数据与缓存层
5. 向量与模型层
6. 部署与代理层

## 3. 总架构图元素清单

### 3.1 浏览器层

- 管理端用户
- 普通用户

### 3.2 前端应用层

## 4. 你画总架构图时可以这样讲

浏览器层发起请求，前端分别进入管理端和用户端。前端请求统一进入 Nginx，再由 Nginx 转发给统一后端。

统一后端内部虽然是一个进程启动，但在代码结构上仍然保留了 `manager` 和 `consumer` 的业务边界。

后端业务执行时会访问 MySQL、Redis、Qdrant，并根据场景调用 Embedding、Rerank 和 Chat 模型服务。

最终整套服务由 Docker Compose 编排，代码发布由 GitHub Actions 自动完成。

## 5. 登录时序图怎么画

建议参与者：

- Browser
- Frontend
- Backend
- Redis
- MySQL

建议顺序：

1. Browser 请求获取验证码
2. Backend 生成验证码
3. Backend 写 Redis
4. Backend 返回验证码图片和 `captchaId`
5. Browser 提交用户名、密码、验证码、`captchaId`
6. Backend 去 Redis 校验
7. Backend 查 MySQL 校验用户
8. Backend 生成 JWT
9. Backend 返回 token

讲法重点：

- Redis 是验证码临时缓存
- MySQL 存的是用户长期数据
- token 是后续所有受保护接口的前提

## 6. 文件入库时序图怎么画

建议参与者：

- Browser
- Frontend
- Backend
- Storage
- Doc Reader
- Embedding API
- Qdrant
- MySQL

建议顺序：

1. 用户上传文件
2. Backend 保存原始文件
3. Backend 解析文档文本
4. Backend 切片
5. Backend 调用 Embedding API
6. Backend 写 Qdrant
7. Backend 写 MySQL 元数据
8. Backend 返回成功

讲法重点：

- 原始文件和向量数据不是一回事
- 文档解析、切片、向量化才是真正的知识入库过程

## 7. 聊天问答时序图怎么画

建议参与者：

- Browser
- Frontend
- Backend
- MySQL
- Embedding API
- Qdrant
- Rerank API
- Chat Model

建议顺序：

1. 用户输入问题
2. 前端先建立 SSE 连接
3. 后端返回 `connected`
4. 前端调用聊天提交接口
5. 后端根据 `appId` 查应用与知识库关系
6. 后端把问题向量化
7. 后端到 Qdrant 召回
8. 如启用 rerank，则调用 Rerank API
9. 后端拼增强 prompt
10. 后端调用 Chat Model
11. 后端通过 SSE 持续推送内容
12. 前端边收边显示
13. 后端落调用记录和聊天记录

讲法重点：

- 建连和发消息是两步
- `connected` 事件是这次修复的关键点
- 真正的回答不是“模型凭空想”，而是结合了召回内容

## 8. 生产部署拓扑图怎么画

建议元素：

- GitHub
- GitHub Actions Runner
- 服务器 `149.88.92.159`
- Docker Compose
- Nginx
- backend
- mysql8
- redis
- qdrant
- embedding-api
- rerank-api

讲法重点：

- 代码提交后先构建，再上传服务器
- 服务器上不是裸进程，而是 Compose 编排
- 对外真正暴露的是 Nginx 和后端健康检查端口

## 9. 故障排查链路图怎么画

建议按“从外到内”画：

1. 页面现象
2. Network 请求
3. Nginx
4. 后端日志
5. Redis / MySQL / Qdrant
6. Embedding / Rerank / Chat 模型服务

讲法重点：

- 不要一上来就怀疑数据库
- 先看页面请求有没有发出去
- 再看代理层和后端是否收到
- 再看依赖服务是否正常

## 10. 这一页最适合录课的总结话术

真正理解一个项目，不是把目录背下来，而是知道请求是怎么一层一层流动的，数据是怎么一层一层落下来的，出问题时又该从哪一层往里查。

只要总架构图、登录时序图、文件入库时序图、聊天时序图、部署拓扑图这 5 张图你能讲明白，这套项目的主体你就已经掌握了。
