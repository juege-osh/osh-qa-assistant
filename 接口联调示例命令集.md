# AI Assistant 接口联调示例命令集

## 1. 文档定位

这份文档是给你录课、联调、排障时直接复制命令用的。

目标：

- 尽量少写临时脚本
- 先把接口链路跑通
- 出问题时快速确认是前端问题还是后端问题

## 2. 基础地址

本地：

```bash
BASE_URL=http://127.0.0.1:9000
```

线上：

```bash
BASE_URL=http://149.88.92.159:19000
```

## 3. 健康检查

```bash
curl "$BASE_URL/"
```

预期：

- 返回 `code=200`
- `data=UP`

## 4. 管理端验证码与登录

### 4.1 获取验证码

```bash
curl "$BASE_URL/manager/manager/getCode"
```

说明：

- 这里只能拿到验证码图片和 `captchaId`
- 真正答案在 Redis 里

### 4.2 登录

```bash
curl -X POST "$BASE_URL/manager/manager/login" \
  -H 'Content-Type: application/json' \
  -d '{
    "username":"admin",
    "pwd":"123456",
    "captchaId":"替换成实际captchaId",
    "code":"替换成实际验证码"
  }'
```

## 5. 用户端验证码、注册、登录

### 5.1 获取验证码

```bash
curl "$BASE_URL/consumer/user/getCode"
```

### 5.2 注册

```bash
curl -X POST "$BASE_URL/consumer/user/register" \
  -H 'Content-Type: application/json' \
  -d '{
    "username":"course_demo_user",
    "pwd":"123456",
    "captchaId":"替换成实际captchaId",
    "code":"替换成实际验证码"
  }'
```

### 5.3 登录

```bash
curl -X POST "$BASE_URL/consumer/user/login" \
  -H 'Content-Type: application/json' \
  -d '{
    "username":"course_demo_user",
    "pwd":"123456",
    "captchaId":"替换成实际captchaId",
    "code":"替换成实际验证码"
  }'
```

说明：

- 登录成功后，把返回里的 `token` 保存下来

## 6. 用户端应用和知识库

假设：

```bash
TOKEN='替换成用户token'
```

### 6.1 创建知识库

```bash
curl -X POST "$BASE_URL/consumer/knowledgeLib/add" \
  -H 'Content-Type: application/json' \
  -H "Authorization: $TOKEN" \
  -d '{
    "libName":"课程演示知识库",
    "libDesc":"用于录课演示文档上传和检索"
  }'
```

### 6.2 查询知识库

```bash
curl -X POST "$BASE_URL/consumer/knowledgeLib/queryPage" \
  -H 'Content-Type: application/json' \
  -H "Authorization: $TOKEN" \
  -d '{
    "pageNow":1,
    "pageSize":10
  }'
```

### 6.3 创建应用

```bash
curl -X POST "$BASE_URL/consumer/app/add" \
  -H 'Content-Type: application/json' \
  -H "Authorization: $TOKEN" \
  -d '{
    "appName":"课程演示助手",
    "appDesc":"用于录课演示问答助手"
  }'
```

### 6.4 查询应用

```bash
curl -X POST "$BASE_URL/consumer/app/queryPage" \
  -H 'Content-Type: application/json' \
  -H "Authorization: $TOKEN" \
  -d '{
    "pageNow":1,
    "pageSize":10
  }'
```

### 6.5 绑定知识库

```bash
curl -X POST "$BASE_URL/consumer/app/bindLib" \
  -H 'Content-Type: application/json' \
  -H "Authorization: $TOKEN" \
  -d '{
    "id":"替换成应用ID",
    "libId":"替换成知识库ID"
  }'
```

## 7. 聊天会话与 SSE

### 7.1 创建会话

```bash
curl -X POST "$BASE_URL/consumer/chat/add" \
  -H 'Content-Type: application/json' \
  -H "Authorization: $TOKEN" \
  -d '{
    "appId":"替换成应用ID",
    "chatName":"课程演示会话"
  }'
```

### 7.2 建立 SSE 连接

```bash
curl -N "$BASE_URL/consumer/chat/connect?chatId=替换成会话ID&token=$TOKEN"
```

预期：

- 先收到 `event: connected`
- 再收到 `data: ok`

### 7.3 发送聊天消息

```bash
curl -X POST "$BASE_URL/consumer/chat/chat" \
  -H 'Content-Type: application/json' \
  -H "Authorization: $TOKEN" \
  -d '{
    "chatId":"替换成会话ID",
    "message":"请介绍一下这个知识库里的核心内容"
  }'
```

## 8. 开放 API 调用示例

接口：

- `POST /consumer/api/chat`

说明：

- 这里不是用登录 token
- 这里要用用户自己的 `appKey`

示例：

```bash
curl -N -X POST "$BASE_URL/consumer/api/chat" \
  -H 'Content-Type: application/json' \
  -d '{
    "appId":"替换成应用ID",
    "appKey":"替换成用户中心里的appKey",
    "chatId":"替换成调用方自定义会话ID",
    "userInput":"请总结知识库内容"
  }'
```

## 10. 最适合录课展示的命令组合

如果你录课时只想展示最关键的“后端并不是黑盒”，建议演示这 5 组命令：

1. 健康检查
2. manager 获取验证码
3. consumer 登录
4. chat/connect 建立 SSE
5. consumer/api/chat 开放接口调用

这样既能讲接口，也能讲 SSE，还能讲开放能力。
