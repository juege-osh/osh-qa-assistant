# AI Assistant 录课前检查清单

## 1. 这份清单的用途

这份清单是给你录课前 10 到 20 分钟使用的。

目标只有一个：

- 把那些最容易让录课中断的问题提前排掉

## 2. 环境检查

- 本地麦克风正常
- 本地屏幕分辨率和字体大小合适
- IDE 已打开项目并完成索引
- 浏览器已登录需要的账号
- 数据库客户端已连接好
- 终端已进入项目目录

## 3. 本地项目检查

- `./start-local.sh` 能正常启动
- `http://127.0.0.1:9000/` 返回 `UP`
- `http://127.0.0.1:8001/` 可访问
- `http://127.0.0.1:9001/` 可访问

## 4. 线上环境检查

- `http://149.88.92.159:19000/` 返回 `UP`
- `http://149.88.92.159/ai-manager/` 可访问
- `http://149.88.92.159/ai-consumer/` 可访问

## 5. 账号检查

- 管理端 `admin / 123456` 可登录
- 用户端演示账号可登录
- 用户端演示账号能在用户中心看到 `appKey`

## 6. 演示数据检查

- 已有 1 个演示知识库
- 已有 1 个演示应用
- 应用已绑定知识库
- 至少已有 1 个上传文件
- 至少已有 1 条历史聊天
- 统计页和调用记录页有数据

## 7. 课程演示链路检查

- 新建知识库可用
- 上传文件可用
- 文件入库可用
- 创建应用可用
- 绑定知识库可用
- 创建会话可用
- 发送消息可用
- SSE 流式返回可用
- OpenAPI 聊天接口可用

## 8. 脚本检查

建议录课前执行一次：

```bash
REDIS_HOST=149.88.92.159 \
REDIS_PORT=6379 \
REDIS_PASSWORD='OshQaRedis@2026' \
CONSUMER_USER=course_demo_user \
CONSUMER_PWD=123456 \
./scripts/smoke-test.sh http://149.88.92.159
```

预期结果：

- 输出 `smoke test passed`

## 9. 故障演示素材检查

- 已确定要演示哪 1 到 2 个故障
- 已准备好对应断点位置
- 已准备好恢复步骤
- 不会影响线上生产主环境

## 11. 录课前最后 3 分钟动作

### 11.1 浏览器标签页准备好

- 用户端
- 管理端
- 数据库客户端
- IDE
- 终端

### 11.2 把无关窗口关掉

- 聊天软件
- 不相关浏览器标签
- 容易弹窗的软件

### 11.3 最后确认一句

今天这节课我是要讲“完整闭环”，不是只演示某一个页面。所以登录、知识库、上传、聊天、统计、调用记录、部署入口，这几块一定都要先确认一遍。
