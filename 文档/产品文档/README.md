# 产品文档目录

> 注意：此文件不可更改，除非用户同意更改，且更改涉及到产品开发的整体方向。

本目录用于存放和维护项目的产品级文档。

这里的内容不等同于开发实现细节，而是用于明确：

- 产品优先服务给谁
- 产品核心价值是什么
- 不同角色的主次优先级
- 产品边界和阶段性取舍
- 会影响整体方向的关键原则

## 使用约束

本目录下的文档属于产品方向基线文档。

后续使用时，必须遵循以下规则：

- 不能基于 AI 自行推断直接修改已敲定的产品结论
- 不能因为实现便利、局部需求或个人理解变化而直接改动产品方向
- 如果发现表述含糊、存在冲突或需要调整，应先获得用户确认
- 只有在用户明确同意的前提下，才可以修改涉及产品整体方向的内容

这里的“产品整体方向”包括但不限于：

- 产品定位
- 用户分层与优先级
- MVP 范围与取舍
- 核心场景定义
- 验收标准口径
