# AI Assistant 断点调试路线图

## 1. 文档定位

这份文档是给你录课或自己排障时直接照着打断点用的。

目标不是把所有类都打满断点，而是用最少的断点把整条业务链看清楚。

## 4. 如果录课时间不够，只打哪 3 个断点

如果你只有很短的时间，我建议只打这 3 个：

1. `UserServiceImpl.login`
2. `AuthorizationInterceptor.preHandle`
3. `AiChatServiceImpl`

因为这 3 个点能把：

- 登录鉴权
- 请求进入系统
- RAG 主链路

三件最核心的事讲透。

## 5. 最适合讲“真实工程味”的断点组合

如果你想让课程不只是“教学 Demo”，而是更像真实项目排障，建议组合是：

1. `AuthorizationInterceptor.preHandle`
2. `ChatServiceImpl`
3. 前端 `Chat.vue`

这组断点最适合讲这次真实修掉的 SSE 无效会话问题。
