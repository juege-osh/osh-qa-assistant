## 2. 先用一句话理解这个项目

这是一个完整的 RAG 问答系统。

你可以把它理解成：

- 用户上传文档
- 系统把文档切片、向量化、存入向量库
- 用户提问时，系统先去知识库里检索相关内容
- 再把检索结果交给大模型生成答案

它不是一个只有聊天框的 Demo，而是一套完整产品，包含：

- 管理后台
- 用户工作台
- 文档上传
- 知识库管理
- 向量检索
- 重排序
- 模型调用
- 调用记录和统计

## 4. 先认识几个最重要的概念

### 4.1 什么是大模型

大模型就是负责“生成回答”的那个模型。

当前项目里用的是：

- Base URL：`https://v2.aicodee.com`
- Model：`MiniMax-M2.7-highspeed`

### 4.2 什么是 Embedding

Embedding 可以理解成“把一句话变成一串向量数字”。

为什么要这样做：

- 文本不能直接做向量相似度比较
- 先转成向量，才能去向量数据库里查“最像”的内容

当前项目里用的是：

- 模型：`BAAI/bge-small-zh-v1.5`

### 4.3 什么是 Rerank

Rerank 可以理解成“二次排序器”。

流程是：

1. 先粗召回一批候选内容
2. 再用 rerank 模型重新排序
3. 让更相关的内容排到前面

当前项目里用的是：

- 模型：`BAAI/bge-reranker-base`

### 4.4 什么是 Qdrant

Qdrant 是向量数据库。

它不负责生成答案，它负责存放向量，并在提问时帮你找相似内容。

### 4.5 什么是 SSE

SSE 是一种服务端实时推送技术。

你在聊天页看到 AI 一点一点往外吐字，就是靠 SSE。

## 8. 这个项目聊天时，内部到底做了什么

你可以把一次问答理解成下面这 8 步：

1. 用户输入问题
2. 后端接收问题
3. 把问题转成向量
4. 去 Qdrant 检索相似知识片段
5. 用 rerank 模型重新排序
6. 拼装 Prompt
7. 调用大模型生成答案
8. 通过 SSE 一段一段推给前端

## 12. 手工部署到底该怎么做

### 12.1 第一步：构建后端

```bash
mvn -f pom.xml -pl backend -am package -DskipTests
```

构建后的包在这里：

- `backend/target/backend.jar`

### 12.2 第二步：构建两个前端

管理端：

```bash
cd frontend/ai-assistant-manager
npm ci --legacy-peer-deps
npm run build
```

用户端：

```bash
cd frontend/ai-assistant-consumer
npm ci --legacy-peer-deps
npm run build
```

### 12.3 第三步：上传文件到服务器

至少要上传这些：

- `backend/target/backend.jar`
- `deploy/server/docker-compose.yml`
- `deploy/server/config/backend-prod-149.88.92.159.yml`
- `deploy/server/nginx.149.88.92.159.conf`
- `frontend/ai-assistant-manager/dist/`
- `frontend/ai-assistant-consumer/dist/`

### 12.4 第四步：准备服务器环境变量

服务器上的 `.env` 负责运行时参数。

重点变量包括：

- `AI_ASSISTANT_DB_PASSWORD`
- `AI_ASSISTANT_REDIS_PASSWORD`
- `AI_ASSISTANT_OPENAI_API_KEY`
- `AI_ASSISTANT_OPENAI_BASE_URL`
- `AI_ASSISTANT_CHAT_MODEL`
- `AI_ASSISTANT_EMBEDDING_MODEL`
- `AI_ASSISTANT_RERANK_MODEL`

### 12.5 第五步：启动服务

在服务器执行：

```bash
cd /opt/qa-assistant
docker compose up -d --remove-orphans
```

### 12.6 第六步：检查服务状态

```bash
docker ps
curl http://127.0.0.1:19000/
```

健康检查正常后，再去浏览器访问：

- `http://149.88.92.159/ai-consumer/`
- `http://149.88.92.159/ai-manager/`

## 14. 新手最容易踩的坑

### 14.1 Java 版本不对

症状：

- Maven 编译报错
- Spring Boot 起不来

处理：

- 必须用 JDK 17

### 14.2 端口被占用

症状：

- 本地启动时报 9000、8001、9001 被占用

处理：

```bash
./stop-local.sh
```

### 14.3 数据库或 Redis 密码不对

症状：

- 后端一启动就连不上数据库或 Redis

处理：

- 检查 `.env.local`
- 检查服务器 `.env`

### 14.4 Embedding 服务没起

症状：

- 上传知识库或发起问答时报向量化失败

处理：

- 先确认 `http://127.0.0.1:18080` 或容器内 `embedding-api:80` 是否可用

### 14.5 Rerank 服务没起

症状：

- 问答链路重排序失败

处理：

- 确认 `http://127.0.0.1:17997` 或容器内 `rerank-api:7997` 是否可用

### 14.6 前端能打开，聊天却报错

最常见原因有 3 类：

1. SSE 没连上
2. 模型配置不对
3. 知识库或应用绑定关系不完整

当前版本已经修复过一个典型问题：

- 聊天页 SSE 原生 `EventSource` 无法发 `Authorization` 请求头
- 现在改为 `query token + 后端拦截器兼容读取`

## 15. 出问题时，先按什么顺序查

推荐顺序：

1. 先看页面能不能打开
2. 再看后端健康检查
3. 再看 Docker 容器状态
4. 再看后端日志
5. 再看模型服务是否健康
6. 最后再查数据库和 Redis

最常用命令：

```bash
docker ps
docker logs --tail 200 qa-assistant-backend
docker logs --tail 200 qa-assistant-embedding-api
docker logs --tail 200 qa-assistant-rerank-api
curl http://127.0.0.1:19000/
```
