# 公开应用 UI 设计规范

本规范基于当前公开应用页已经落地的视觉风格整理，后续新增页面默认先读取本文件，再开始设计与实现，保证整体风格统一。

适用关键词：
- 干净
- 专业
- 克制
- 轻量 Stripe 风格
- 浅色、低噪音、高可读性

## 1. 配色规范

### 1.1 核心色板

| 角色 | 变量 | 色值 | 用途 |
| --- | --- | --- | --- |
| 主色 | `--public-accent` | `#635BFF` | 主要操作、高亮状态、品牌识别 |
| 主色加深 | `--public-accent-strong` | `#4B44E8` | 主按钮渐变终点、悬停强化 |
| 主色浅底 | `--public-accent-soft` | `#EFF1FF` | 轻提示、状态底、AI 标识底色 |
| 成功色 | `--public-success` | `#12B76A` | 成功反馈，谨慎使用 |
| 页面背景 | `--public-bg` | `#F6F9FC` | 页面主背景 |
| 背景辅助色 | `--public-bg-soft` | `#EEF3F8` | 区块层次、弱分隔 |
| 面板白 | `--public-panel` | `#FFFFFF` | 卡片、容器基础底色 |
| 面板强白 | `--public-panel-strong` | `#FFFFFF` | 主要面板、主内容区 |
| 面板浅蓝白 | `--public-panel-muted` | `#F9FBFF` | 输入区、次级面板、轻顶栏 |
| 默认边框 | `--public-border` | `#E6EBF2` | 卡片、输入框、按钮边框 |
| 强边框 | `--public-border-strong` | `#D7DFEB` | hover、强调边界 |
| 主文字 | `--public-text` | `#101828` | 标题、重点信息 |
| 次主文字 | `--public-text-soft` | `#344054` | 正文、次级标题 |
| 弱文字 | `--public-text-muted` | `#667085` | 时间、说明、标签 |

### 1.2 背景使用规则

- 页面整体背景使用浅灰蓝系渐变，不使用纯白铺满整个页面。
- 主内容容器使用白色或浅蓝白，避免过重的灰底。
- 同一页面最多保留一套主强调色，不引入第二套抢眼高饱和颜色。
- 大面积区域优先用“底色变化”做层级，不优先用重阴影或重描边。

### 1.3 用色原则

- 主色只用于主按钮、关键状态、焦点操作，不要全页泛滥。
- 弱提示优先用 `#EFF1FF` 这类浅底承载，而不是直接上高饱和紫。
- 开发提示、技术说明、策略说明不进入用户页面。

## 2. 字体规范

当前项目延续现有字体体系，不额外引入新字体族，重点统一字号、字重、行高。

### 2.1 标题层级

| 层级 | 建议字号 | 字重 | 行高 | 用途 |
| --- | --- | --- | --- | --- |
| 一级标题 | `20px` | `700` | `1.3` | 欢迎标题、核心页面标题 |
| 二级标题 | `18px` | `700` | `1.35` | 密码验证区标题、模块标题 |
| 三级标题 | `14px` | `700` | `1.2-1.4` | 应用名称、消息块标题、卡片标题 |
| 辅助标题 | `14px` | `600` | `1.4` | 区域名、会话标题、工具栏标题 |

### 2.2 正文层级

| 层级 | 建议字号 | 字重 | 行高 | 用途 |
| --- | --- | --- | --- | --- |
| 主正文 | `15px` | `400` | `1.6-1.86` | 聊天正文、主要内容说明 |
| 常规正文 | `14px` | `400/600` | `1.65-1.75` | 按钮文字、卡片内容、会话名称 |
| 辅助正文 | `13px` | `400` | `1.65-1.7` | 空态说明、验证说明 |
| 标签/状态 | `12px` | `500/600` | `1.4-1.5` | 状态胶囊、时间、脚注、提示 |
| 极小标签 | `11px` | `600` | `1.4` | chip、微状态、流式标记 |

### 2.3 文字风格规则

- 标题统一偏重，正文保持轻，靠字重差异拉开层级。
- 正文颜色默认使用 `#344054`，不要直接大段使用纯黑。
- 说明性文字统一使用 `#667085`，减少信息噪音。
- 标题允许轻微负字距，当前主欢迎标题使用 `letter-spacing: -0.02em`。
- 用户可见文案必须产品化，不出现开发备注、实现提示、策略说明。

## 3. 间距规范

### 3.1 基础间距刻度

推荐统一使用以下间距刻度：

| Token | 数值 | 用途 |
| --- | --- | --- |
| `space-4` | `4px` | 极小内间距、微调 |
| `space-6` | `6px` | chip 间距、图标组 |
| `space-8` | `8px` | 图标与文字、状态与标题 |
| `space-10` | `10px` | 小按钮组、会话操作区 |
| `space-12` | `12px` | 小模块间距、消息元信息 |
| `space-14` | `14px` | 卡片内部 padding、小屏块间距 |
| `space-16` | `16px` | 主要内边距、区块基础间距 |
| `space-18` | `18px` | 聊天气泡、面板内间距 |
| `space-20` | `20px` | 页面主要区块、卡片舒适内边距 |
| `space-24` | `24px` | 主内容区 padding、模块间距 |

### 3.2 页面与模块间距

- 页面外边距：
  - 桌面端 `20px 22px 24px`
  - 移动端 `12px`
- 主布局列间距：
  - 桌面端 `16px`
  - 窄屏 `14px`
- 主面板内部常用 padding：
  - 侧栏 `18px`
  - 聊天内容区 `24px`
  - 输入区 `14px 16px`
  - 顶栏行高区 `14px 0`
- 区块上下节奏：
  - 聊天消息块底部 `24px`
  - 欢迎区下方 `16px`
  - 说明脚注顶部 `10px`

### 3.3 留白原则

- 优先通过 padding 和区块间距制造呼吸感，不靠空文案占空间。
- 输入区、消息区、侧栏都要有稳定边界，不出现拥挤堆叠。
- 同级模块优先使用统一间距，不同层级再做放大。

## 4. 组件样式规范

## 4.1 圆角规则

| 组件 | 圆角 |
| --- | --- |
| 主面板 / 大容器 | `20px` |
| 输入区 / 控件容器 | `18px` |
| 普通按钮 | `12px` |
| 会话卡片 / 欢迎卡片 / 对话气泡 | `16px-18px` |
| 小图标按钮 | `10px` |
| 胶囊状态 / chip | `999px` |

规则：
- 页面整体是“柔和圆角”体系，不使用硬直角。
- 胶囊只用于状态、标签、轻操作，不扩散到所有组件。

## 4.2 阴影规则

| 名称 | 值 | 用途 |
| --- | --- | --- |
| 主阴影 | `0 12px 32px rgba(15, 23, 42, 0.04)` | 重要悬浮卡片、主按钮 hover |
| 轻阴影 | `0 4px 12px rgba(15, 23, 42, 0.03)` | 主面板、输入区、普通卡片 |
| 主按钮阴影 | `0 10px 18px rgba(99, 91, 255, 0.18 ~ 0.20)` | 主 CTA |
| 激活卡片阴影 | `0 8px 18px rgba(99, 91, 255, 0.08)` | 当前选中会话 |

规则：
- 阴影必须轻，偏空气感，不要厚重投影。
- 纯黑重阴影禁用。
- 大部分层级关系优先靠边框和背景层次，不靠强阴影。

## 4.3 边框规则

- 默认边框：`1px solid #E6EBF2`
- 强调边框：`1px solid #D7DFEB`
- 激活态边框：带主色透明度，例如 `rgba(99, 91, 255, 0.26 ~ 0.34)`
- 危险态边框：浅红边，例如 `#FECACA`
- 空态占位允许使用虚线边框，例如 `1px dashed #D7DFEB`

规则：
- 边框是当前风格的主要结构语言之一。
- 不要一个区域内“外层一圈、内层再一圈”重复套边框。
- 顶栏这类结构条优先用铺底色加底部分隔线，不再额外套卡片框。

## 4.4 按钮规范

### 主按钮

- 高度：`40px`，大型主要入口可到 `48px`
- 圆角：`12px`
- 背景：`linear-gradient(180deg, #635BFF 0%, #4B44E8 100%)`
- 文字：`#FFFFFF`
- 字号：`14px`
- 字重：`600`
- 阴影：`0 10px 18px rgba(99, 91, 255, 0.18 ~ 0.20)`

适用：
- `发送`
- `开启新对话`
- 关键确认动作

### 次级按钮

- 高度：`34px` 或 `40px`
- 背景：白色或浅白底
- 边框：`1px solid #E6EBF2`
- 文字：`#344054`
- 圆角：顶栏动作用 `999px`，常规操作按钮用 `12px`

适用：
- `应用信息`
- `进入工作台`
- `清空`

### 悬停与禁用

- hover：整体轻微上浮 `translateY(-1px)`，边框加深，阴影略增强
- disabled：透明度下降到约 `0.55`，取消阴影，鼠标改为 `not-allowed`

## 4.5 卡片规范

### 面板卡片

- 背景：白色或浅蓝白渐变
- 边框：`1px solid #E6EBF2`
- 圆角：`20px`
- 阴影：轻阴影

适用：
- 侧栏
- 聊天主面板
- 输入区容器

### 内容卡片

- 圆角：`16px`
- 内边距：`16px ~ 20px`
- 背景：`#FFFFFF`
- hover：轻微上浮，不做剧烈动效

适用：
- 欢迎建议卡
- 会话记录卡

### 对话气泡

- AI 气泡：
  - 白底
  - `1px solid #E6EBF2`
  - `18px` 圆角
  - 轻阴影
- 用户气泡：
  - 主色渐变底
  - 白字
  - 带主色透明边框

## 4.6 顶栏规范

- 顶栏是结构区，不是独立卡片。
- 样式：
  - 铺满所在整行
  - 浅蓝白底色，和输入区同系
  - 只保留底部分隔线
  - 不额外套独立边框
- 左侧放应用识别信息
- 右侧放 1~2 个轻操作按钮

## 4.7 输入区规范

- 容器圆角：`18px`
- 容器背景：`linear-gradient(180deg, #FFFFFF 0%, #FBFCFF 100%)`
- 边框：`1px solid #E6EBF2`
- 文本域最小高度：`56px`
- 文本域本体无额外描边，保持透明嵌入容器
- 输入下方只保留和发送相关的状态、操作、脚注

## 4.8 滚动条规范

- 优先封装为公共类统一复用，当前公共类名：`space-scrollbar`
- 适用区域：
  - 侧栏会话列表
  - 聊天内容滚动区
  - 后续新增的浅色滚动容器
- 风格要求：
  - 细滚动条，不使用系统默认粗灰条
  - 轨道透明
  - 滑块为偏实的灰蓝色，hover 时略加深
  - 滑块必须是胶囊圆角
  - 优先悬浮在容器内侧留白区，不挤占正文或列表卡片的有效宽度
- 推荐实现参数：
  - 滚动条宽高：`10px`
  - 滑块边框：`2px solid transparent`
  - 滑块底色：`rgba(148, 160, 180, 1)`
  - hover 底色：`rgba(122, 136, 157, 1)`
  - Firefox 使用 `scrollbar-width: thin`

## 5. 布局与统一性原则

- 页面风格统一为“浅色、轻边框、低噪音、强可读性”。
- 不叠加玻璃遮罩、厚模糊、重渐变、复杂装饰。
- 功能入口尽量直给，避免为两个按钮再包一层“更多”。
- 同类页面优先复用本规范里的 token，不重新发明配色和圆角。
- 设计始终面向用户，不向页面暴露开发逻辑和内部备注。

## 6. 新页面执行要求

后续新增页面默认遵循以下流程：

1. 先阅读本文件 `design.md`
2. 优先复用本文件中的颜色、字号、间距、圆角、边框、阴影
3. 如果有特殊页面需要偏离，先说明偏离原因，再做局部调整
4. 没有充分理由时，不新增第二套视觉语言

如果后续这套风格继续演进，以本文件为准同步更新。
